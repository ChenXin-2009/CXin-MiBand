# 数据库体积分析总结

## 🔍 核心发现

### 问题根源：索引占用了 80% 的空间！

```
数据库总大小: 811.85 MB
├─ 字段数据: 163.66 MB (20.16%)
└─ 索引+元数据: 648.19 MB (79.84%) ⚠️
```

**这是最关键的发现！** 大部分空间被索引占用，而不是数据本身。

## 📊 详细分析

### 1. 字段体积占用（前5名）

| 排名 | 字段 | 大小 | 占比 | 说明 |
|------|------|------|------|------|
| 1 | word | 49.03 MB | 6.04% | 必须保留 |
| 2 | sw | 45.35 MB | 5.59% | 可删除（用word替代） |
| 3 | translation | 40.43 MB | 4.98% | 必须保留（需压缩） |
| 4 | definition | 16.23 MB | 2.00% | 可删除 |
| 5 | exchange | 6.36 MB | 0.78% | 可删除 |

**字段数据总计**: 163.66 MB

### 2. 索引情况

数据库有 **6 个索引**，占用 **648.19 MB**：

```sql
1. sqlite_autoindex_stardict_1 (自动)
2. sqlite_autoindex_stardict_2 (自动)
3. stardict_1 - UNIQUE INDEX on id
4. stardict_2 - UNIQUE INDEX on word
5. stardict_3 - INDEX on (sw, word collate nocase)
6. sd_1 - INDEX on (word collate nocase)
```

### 3. Translation 内容分析

- 23.55% 包含 `[网络]` 标记 (801,448 词)
- 4.43% 包含 `[地名]` 标记 (150,806 词)
- 0.81% 包含 `[人名]` 标记 (27,561 词)
- 平均长度: 12.51 字符
- 最大长度: 11,814 字符
- 超过 500 字符: 1,125 词

## 💡 优化方案

### 三大优化策略

#### ✅ 策略 1：减少索引（节省 ~640 MB）
- 删除 5 个索引
- 只保留 word 的主键索引
- **这是最重要的优化！**

#### ✅ 策略 2：筛选词汇（节省 ~800 MB）
- 从 3,402,564 词减少到 20,000 词
- 保留：牛津3000 + 柯林斯星级 + BNC高频词
- 减少 99.4% 的词条

#### ✅ 策略 3：精简字段（节省 ~70 MB）
- 删除：sw, definition, exchange, detail, audio, tag, pos
- 保留：word, phonetic, translation, collins, oxford
- 从 15 个字段减少到 5 个

#### ✅ 策略 4：压缩内容（节省 ~20 MB）
- 删除 translation 中的标记
- 限制最大长度 200 字符
- 减少 50-70% 的翻译内容

## 🎯 最终方案

### 推荐配置

```sql
CREATE TABLE dict (
  word TEXT PRIMARY KEY,      -- 单词
  phonetic TEXT,              -- 音标
  translation TEXT,           -- 翻译（压缩）
  collins INTEGER DEFAULT 0,  -- 柯林斯星级
  oxford INTEGER DEFAULT 0    -- 牛津3000
);
```

### 筛选条件

```sql
WHERE oxford > 0 
   OR collins > 0 
   OR (bnc > 0 AND bnc <= 20000)
```

## ✨ 关键要点

1. **索引是主要问题**：占用了 80% 的空间
2. **重建数据库**：不要在原数据库上修改，要创建新的精简数据库
3. **只建一个索引**：word 字段的主键索引就够了
4. **词汇筛选很重要**：20,000 词足够日常使用
5. **压缩 translation**：删除标记和冗余内容
