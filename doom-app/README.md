# 毁灭吧 (Doom App)

一个复古风格的第一人称射击游戏，专为小米手环/手表 Vela OS 开发。

## ✨ 主要特性

### 🎮 经典游戏玩法
- **第一人称视角**：3D 射线投射渲染引擎
- **敌人战斗**：消灭所有敌人才能通关
- **物资收集**：拾取弹药和血包补给
- **迷宫探索**：在迷宫中找到出口

### 🕹️ 操控方式
- **前进/后退**：移动角色
- **左转/右转**：旋转视角
- **左移/右移**：横向移动（扫射）
- **开火**：射击敌人
- **长按支持**：按住按钮持续执行动作

### 🎨 视觉效果
- **3D 渲染**：基于射线投射的伪3D效果
- **动态光影**：根据距离调整墙壁明暗
- **枪械动画**：开火时的后坐力和枪口火焰
- **敌人精灵**：简化的火柴人敌人
- **准星瞄准**：屏幕中央的十字准星

### 💡 游戏机制
- **血量系统**：被敌人攻击会减少血量
- **弹药管理**：子弹有限，需要拾取补给
- **碰撞检测**：无法穿墙，敌人近距离会攻击
- **振动反馈**：开火和受伤时的触觉反馈

## 📱 游戏界面

### HUD（抬头显示）
- **血量条**：显示当前生命值
- **弹药数**：显示剩余子弹
- **敌人数**：显示剩余敌人数量
- **消息提示**：显示游戏状态（找出口、命中、受伤等）

### 游戏画面
- **天空**：深蓝色背景
- **地板**：棕色地面，带有距离渐变
- **墙壁**：多色墙壁，根据区域变化
- **敌人**：红色火柴人
- **物品**：弹药（黄色）、血包（绿色）、出口（青色）

### 控制面板
- **6个按钮**：前进、左转、右转、左移、右移、开火
- **颜色区分**：主要操作（蓝色）、次要操作（灰色）、危险操作（红色）

### 游戏规则说明
- 使用方向键移动和转向
- 点击开火按钮射击敌人
- 收集弹药和血包补给
- 消灭所有敌人后找到出口
- 注意保持血量，避免被敌人攻击

## 🚀 快速开始

### 环境要求
- Node.js >= 14.0.0
- npm >= 6.0.0
- 小米手环/手表（支持 Vela OS）
- 最低平台版本：1070

### 安装依赖
```bash
cd doom-app
npm install
```

### 开发调试
```bash
# 开发模式（带热更新）
npm run dev

# 或使用 AIoT-toolkit
npx aiot-toolkit start
```

### 构建发布
```bash
# 构建项目
npm run build

# 打包发布
npm run release
```

打包后的 .rpk 文件位于：`dist/com.doom.band.debug.0.1.0.rpk`

### 安装到手环
1. 将打包好的.rpk文件传输到手机
2. 打开小米运动健康App
3. 进入"设备" → 选择手环 → "应用管理"
4. 点击"添加应用"，选择.rpk文件
5. 等待安装完成

## 📖 游戏指南

### 基本操作

1. **移动**
   - 点击"前进"按钮向前移动
   - 长按可以持续移动
   - 使用"左移"/"右移"进行横向移动

2. **转向**
   - 点击"左"/"右"按钮旋转视角
   - 长按可以持续旋转
   - 找到敌人或出口的方向

3. **战斗**
   - 将准星对准敌人
   - 点击"开火"按钮射击
   - 注意弹药数量，及时拾取补给

4. **生存**
   - 远离敌人避免受伤
   - 拾取血包恢复生命值
   - 血量归零会游戏失败

5. **通关**
   - 消灭所有敌人
   - 找到出口（青色标记）
   - 走到出口位置即可通关

### 游戏技巧

1. **战术移动**：使用横向移动（扫射）可以在射击时躲避敌人
2. **弹药管理**：优先拾取弹药，避免无弹可用
3. **血量控制**：保持安全距离，血量低时优先拾取血包
4. **地图记忆**：记住地图布局，快速找到物资和出口
5. **瞄准技巧**：敌人在准星附近即可命中，不需要完全对准

### 渲染技术

**射线投射（Raycasting）**
- 从玩家位置向视野方向发射射线
- 检测射线与墙壁的交点
- 根据距离计算墙壁高度和明暗
- 32列射线构成完整视野

**精灵渲染**
- 计算敌人/物品相对玩家的角度和距离
- 判断是否在视野范围内
- 根据距离缩放精灵大小
- 绘制在对应的屏幕位置

**地板渲染**
- 预定义6个距离采样点
- 根据视角方向采样地板颜色
- 绘制水平色带模拟地板

### 性能优化

- **逻辑跳帧**：每3帧执行一次游戏逻辑
- **精灵跳帧**：每2帧更新一次精灵位置
- **帧率限制**：目标16ms/帧（约60fps）
- **最大时间步**：限制70ms避免大跳跃

## ⚠️ 注意事项

1. **电量消耗**：游戏持续运行会消耗电量
2. **操作延迟**：快速点击可能有延迟，建议使用长按
3. **难度设置**：当前版本难度固定，敌人数量和位置固定
4. **存档功能**：暂不支持存档，每次都是新游戏
5. **地图限制**：当前只有一个关卡

## 🎮 游戏参数

### 可调整参数（开发者）

```javascript
// 移动速度
MOVE_SPEED = 0.0022      // 前进速度
STRAFE_SPEED = 0.0018    // 横移速度
TURN_SPEED = 0.0026      // 转向速度

// 战斗参数
FIRE_INTERVAL = 180      // 射击间隔（ms）
ENEMY_DAMAGE = 4         // 敌人伤害
ENEMY_HP = 2             // 敌人血量

// 视野参数
FOV = Math.PI / 3        // 视野角度（60度）
MAX_DEPTH = 9            // 最大渲染距离
COLS = 32                // 射线数量
```
